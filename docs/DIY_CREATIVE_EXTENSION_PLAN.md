# DIY Creative Extension Plan

## 当前实现判断

- DIY 入口集中在 `public/index.html` 的视觉控制台 `#fx-panel`。
- “我的作品”继续复用 `mineradio-user-fx-archives-v1` 保存完整视觉配置，支持复制、导入和导出 `.mineradio-visual.json`。
- “形态工坊”已经接入真实 Three.js 预览和主播放器渲染，用户可以用基础粒子创建自己的 3D 形态。
- 自定义形态保存为 `shapeRecipe`，本地存储在 `mineradio-shape-presets-v1`，当前应用形态存储在 `mineradio-active-shape-preset-v1`。
- 自定义形态支持导入/导出 `.mineradio-shape.json`。
- 公开版保留通用基础粒子和结构能力，不内置二创作者个人实验形态。

## 已落地能力

- 基础粒子：点、线、环、曲线环、面、螺旋、波、隧道、球面、星尘、卡片等。
- 基础结构包：唱片结构、星河结构、隧道结构、球面结构、照片墙结构、粒子帘结构。
- 编辑流程：打开形态工坊、添加粒子、选择图层、拖动位置、拖动旋转、用滑条细调参数。
- 预览流程：默认静态编辑，手动切换到律动预览；有歌曲播放时跟随真实音频，没有歌曲时使用演示节拍。
- 图层管理：单看、隐藏/显示、全部预览、删除图层。
- 历史操作：撤销、重做。
- 稳定底座：漂移锁定和一键稳定，用于减少多层形态随音乐律动后慢慢偏移的问题。
- 封面素材：形态内部素材和整屏背景媒体已经拆分，上传形态素材不会自动铺满整屏背景。
- 播放器接入：保存后的“我的形态”会出现在形态预设网格中，可以像内置预设一样应用到播放器。

## 当前边界

- 不开放任意 JavaScript 执行。
- 不开放用户直接粘贴完整 shader 代码。
- 不承诺用户导入的任意形态都能达到内置预设的视觉质量。
- 不内置二创作者个人实验形态；这类形态应作为本地个人作品或单独分享的 `.mineradio-shape.json`。
- 自定义形态仍使用安全参数范围，避免超大粒子数、非法颜色或过高动画强度拖垮性能。

## 后续建议

### 第一阶段：打磨可用性

- 增加更清楚的空状态和新手引导。
- 为常见形态提供“结构包”，但避免把个人实验作品做成默认公开预设。
- 给导入的 `.mineradio-shape.json` 增加版本提示、作者说明和预览信息。
- 改进多个图层重叠时的选择体验。

### 第二阶段：形态分享格式

- 完善 `shapeRecipe` schema，明确字段、版本和兼容性。
- 允许形态文件携带说明、作者名、预览图和推荐音乐类型。
- 导入时先预览，不自动覆盖当前形态。

### 第三阶段：作品化生态

- 将视觉作品和形态预设打通，让一个作品可以引用一个自定义形态。
- 维护一组经过审核的公开示例包。
- 示例包只放通用、可维护的形态，不放个人临时实验稿。
