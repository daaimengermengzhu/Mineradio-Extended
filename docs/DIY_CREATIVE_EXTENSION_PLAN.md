# DIY Creative Extension Plan

## 当前实现判断

- 当前 DIY 入口集中在 `public/index.html` 的视觉控制台 `#fx-panel`。
- 核心配置是全局 `fxDefaults` / `fx`，通过 `localStorage` 的 `mineradio-lyric-layout-v1` 保存。
- 用户存档使用 `mineradio-user-fx-archives-v1`，本质是保存当前 `fx` 参数快照，支持 JSON 导入导出。
- 粒子主体基于 Three.js `BufferGeometry` + `ShaderMaterial` + `Points`。主要形态写在顶点 shader 里，通过 `uPreset` 切换内置预设。
- 当前内置预设为：`emily专辑封面`、`滚筒`、`星球`、`虚空`、`唱片`、`星河`、`安魂`。
- 当前用户能调整的范围主要是：颜色、背景媒体、律动强度、景深、封面清晰度、镜头晃动、歌词样式、叠加效果、桌面歌词、3D 歌单架和少量粒子高级参数。

## 当前限制

- 用户不能创建新的粒子形态，只能在现有预设上调参数。
- 粒子数量、分布方式、形态算法、音频频段绑定和转场逻辑大多写死在 shader 或渲染循环里。
- 存档保存的是已有参数，不是一个完整“作品”概念，缺少作品名、封面、作者描述、预览图、版本和兼容性说明。
- 如果直接开放用户写 JavaScript 或 shader，会有安全、卡顿和崩溃风险，不适合作为第一阶段。

## 建议扩展路线

### 第一阶段：把现有自由度产品化

- 把“用户存档”升级成“我的作品”，增加作品名、说明、预览图、导入、导出和复制为新作品。
- 把已经存在但分散的参数分组为：背景、粒子、歌词、镜头、歌单架、性能。
- 增加更直观的粒子参数：密度、扩散范围、层次深度、音频跟随强度、粒子透明度、光晕大小。
- 所有参数继续走 `fx`，不改底层渲染架构，风险最低。

### 第二阶段：增加声明式粒子配方

- 新增 `creativeRecipe` 配置对象，用 JSON 描述作品，而不是执行用户代码。
- 可开放的配方字段包括：基础形状、层数、颜色来源、速度、扩散、音频绑定、转场方式。
- 配方导入时必须做范围校验，避免超大粒子数、非法颜色或过高动画强度拖垮性能。

### 第三阶段：做作品市场/分享格式

- 作品导出为 `.mineradio-visual.json`。
- 导入时显示来源、版本、风险提示和预览，不自动覆盖当前作品。
- 可在仓库里维护一组扩展版示例作品包，用户下载后直接导入。

## 不建议第一阶段做的事

- 不建议开放任意 JavaScript 执行。
- 不建议让用户直接粘贴完整 shader 代码。
- 不建议一次性把 `public/index.html` 拆成大工程重构。
- 不建议移除现有 `fx` 存储结构，否则会破坏老用户配置。

## 下一步最小实现

1. 先把“用户存档”改名并扩展为“我的作品”。
2. 增加作品导出文件名、预览图、描述字段。
3. 增加几个安全粒子参数，优先接入现有 shader 已支持或容易支持的字段。
4. 做导入校验和版本号，为后续 `creativeRecipe` 留字段。
