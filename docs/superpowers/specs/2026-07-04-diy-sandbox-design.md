# Mineradio Extended DIY 沙盒设计

日期：2026-07-04

## 目标

把 Mineradio Extended 的 DIY 从“切换几个固定视觉预设”升级为“用户可以创造自己的形态预设”。

这里的“沙盒”不是开放任意代码执行，而是开放一组安全、可组合、可保存的粒子基础能力。用户可以像搭乐高一样，用基础例子拼出自己的形态结构，再让歌曲封面、上传图片、文字和歌词作为底片填充到这个形态里。

## 用户确认的方向

采用方案 B：模块化 DIY 沙盒，但需要把“形态预设”和“底片素材”分开。

核心判断：

- 形态预设先存在，它定义粒子怎么排、怎么动、怎么组织。
- 底片素材后填充，它可以是当前歌曲封面、上传图片、文字、歌词或封面取色。
- 素材不决定形态本身，只影响形态的颜色、纹理、内容和局部细节。
- 用户先用基础例子拼出形态，再把形态保存成“我的形态预设”。
- 作品可以记录“形态预设 + 底片来源 + 参数快照”，并支持命名、保存、复制、导入和导出。

## 第一阶段范围

第一阶段目标是让用户明显感受到“我可以创造形态”，同时不破坏现有播放器稳定性。

### 1. 基础例子库

基础例子是最小颗粒，不是完整效果。第一阶段先提供这些可组合元素：

- 点：单个粒子或粒子簇。
- 线：粒子连线或光线。
- 环：圆形轨道和唱片边缘。
- 面：平面网格、幕布、照片底板。
- 螺旋：按角度和半径旋转排布。
- 波：上下起伏或音频驱动的曲面。
- 隧道：沿深度方向推进的环阵列。
- 卡片：带图片纹理的小平面。

这些基础例子可以被用户添加、删除、排序、复制和调参。

### 2. 基础结构库

基础结构是我们先做好的组合模板，用户不用从零拼也能直接用：

- 唱片结构：环 + 中心面 + 旋转。
- 星河结构：螺旋 + 深度 + 发光点。
- 隧道结构：环阵列 + 前后推进。
- 照片墙结构：卡片网格 + 漂浮。
- 粒子帘结构：面 + 波浪 + 音频律动。

基础结构本身也由基础例子组成，后续可以拆开继续改。

### 3. 自定义形态预设

用户可以从基础结构开始，也可以直接从基础例子开始，拼出自己的形态预设。

形态预设保存的是形态规则，不保存某一首歌的具体封面内容。它应该包含：

- 使用了哪些基础例子。
- 每个基础例子的数量、大小、深度、旋转、透明度。
- 每个基础例子的排布方式。
- 哪些参数跟随音频律动。
- 镜头和转场规则。
- 底片如何填充到形态里，例如作为纹理、取色、粒子采样或背景光。

### 4. 底片素材

底片素材只负责填充和演示，不负责定义形态。第一阶段支持：

- 当前歌曲封面。
- 本地上传图片。
- 文字或短句。
- 当前歌词片段。

同一个形态预设切换不同歌曲时，应能用新歌曲封面重新展示。

### 5. 安全参数

用户可调参数必须有范围限制：

- 粒子密度
- 扩散范围
- 层次深度
- 音频跟随强度
- 粒子透明度
- 光晕大小
- 镜头晃动
- 动画速度

这些参数继续走现有 `fx` 配置体系，避免第一阶段大改底层架构。

## 不做什么

第一阶段明确不做：

- 不开放任意 JavaScript 执行。
- 不允许用户直接粘贴完整 shader 代码。
- 不一次性重构整个 `public/index.html`。
- 不移除现有 `fx` 和 `mineradio-user-fx-archives-v1` 兼容逻辑。
- 不把上传图片传到外部服务器；图片应保存在本地浏览器存储或本地文件链路里。
- 不把“上传图片生成形态”作为核心方向；上传图片只是底片，不是形态本体。

## 推荐实现路径

### 第一步：概念纠偏

先把 UI 和文案从“素材/预设简单组合”纠正为“形态工坊”：

- 形态预设区增加“创建形态”入口。
- 新入口明确区分“形态规则”和“底片素材”。
- 已经做出的“基础积木”面板暂时定位为作品组合台，不代表最终的形态工坊。

成功标准：

- 用户看到入口时能理解：这里是在创造形态，不是在换图片。
- 切换歌曲封面时，自定义形态仍然存在。
- 底片来源不会被误认为形态来源。

### 第二步：基础例子库

实现一批可添加的基础例子：

- 点、线、环、面、螺旋、波、隧道、卡片。
- 每个例子有最小参数面板。
- 支持添加、删除、复制、排序。

成功标准：

- 用户能用 2-3 个基础例子拼出和现有预设不同的形态。
- 参数变更能实时预览。
- 参数范围被限制，不会拖垮渲染。

### 第三步：声明式形态配方

新增 `shapeRecipe` 字段，用 JSON 描述形态规则，不执行用户代码。

建议结构：

```json
{
  "type": "mineradio-shape-preset",
  "version": 1,
  "name": "My Shape Preset",
  "material": {
    "role": "fill",
    "defaultSource": "current-cover"
  },
  "primitives": [
    {
      "type": "ring",
      "count": 240,
      "radius": 1.2,
      "depth": 0.15
    },
    {
      "type": "spiral",
      "count": 420,
      "turns": 2.8,
      "audioFollow": 0.58
    }
  ],
  "motion": {
    "cameraMode": "slow-orbit",
    "rotation": 0.32
  }
}
```

成功标准：

- 导入前显示形态预览和来源说明。
- 导入失败有明确提示。
- 超出范围的参数会被拒绝或钳制到安全范围。
- 导入配方不能执行任意代码。

## 交互原则

- 普通用户看到的是“基础例子”“基础结构”和“我的形态”，不是技术参数堆叠。
- 高级用户可以导入导出 JSON，但仍不能执行代码。
- 上传图片是底片来源，不替代当前专辑封面默认逻辑，也不直接定义形态。
- 所有改动优先兼容现有视觉预设，不把已有好看的效果改坏。

## 验证方式

后续实现时至少验证：

- `git diff --check`
- `node --check server.js`
- 前端内联脚本语法检查
- 浏览器打开播放器主页面
- 浏览器打开 DIY 面板并保存、恢复一个作品
- 导入错误 JSON 时不覆盖现有作品

## 当前结论

已确认采用“形态工坊”方向。下一步应先把当前第一阶段实现从“基础积木组合台”纠偏为“形态预设创建入口”，再逐步接入基础例子库和声明式形态配方。

## 2026-07-04 第一阶段实现记录

- 视觉控制台新增“基础积木”入口，覆盖素材、形态、粒子层、律动、镜头和作品保存。
- 素材积木复用现有背景媒体上传链路；默认仍是当前歌曲封面，上传图片或视频不会传到外部服务。
- 形态积木先复用现有稳定预设，避免第一阶段直接改 shader 形态算法。

- “用户存档”界面升级为“我的作品”，保留旧存储键以兼容老用户配置，并新增复制与 `.mineradio-visual.json` 导出。

## 2026-07-04 方向纠偏记录

用户进一步确认：DIY 的核心不是“上传图片生成形态”，也不是“在现有形态边缘调几个参数”，而是让用户用基础例子拼出自己的形态预设。

纠偏后的模型：

- 基础例子：点、线、环、面、螺旋、波、隧道、卡片。
- 基础结构：由多个基础例子组合出的好用模板，例如唱片、星河、隧道、照片墙。
- 自定义形态预设：用户自由组合基础例子和结构后保存下来的形态规则。
- 底片素材：歌曲封面、上传图片、文字和歌词，只负责填充、取色、纹理和演示，不定义形态。

之前已落地的“基础积木”面板可保留为临时作品组合台，但不应被当作最终的高自由度 DIY 形态工坊。
