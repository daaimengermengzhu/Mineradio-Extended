# Mineradio Extended DIY 沙盒设计

日期：2026-07-04

## 目标

把 Mineradio Extended 的 DIY 从“切换几个固定视觉预设”升级为“用户可以用基础积木搭建自己的视觉作品”。

这里的“沙盒”不是开放任意代码执行，而是开放一组安全、可组合、可保存的视觉基础能力。用户可以像搭积木一样组合素材、形态、粒子层、律动、镜头和文字，最后保存为自己的作品。

## 用户确认的方向

采用方案 B：模块化 DIY 沙盒。

核心判断：

- 素材默认继续使用当前歌曲的专辑封面。
- 专辑封面不只作为“唱片图”，也可以作为取色、粒子采样、纹理和背景光场来源。
- 后续允许用户上传自己的图片，作为作品素材来源之一。
- 用户不需要写代码，只通过基础积木组合效果。
- 作品可以命名、保存、复制、导入和导出。

## 第一阶段范围

第一阶段目标是让用户明显感受到自由度提升，同时不破坏现有播放器稳定性。

### 1. 我的作品

把现有“用户存档”升级为“我的作品”：

- 作品名
- 作品说明
- 预览图
- 使用的基础形态
- 使用的素材来源
- 参数快照
- 复制为新作品
- 导入和导出

导出格式后续建议为 `.mineradio-visual.json`。

### 2. 基础积木

第一阶段先开放这些积木：

- 素材：当前专辑封面、本地上传图片
- 形态：唱片场、星云场、瀑布场、照片墙、漂浮卡片
- 粒子层：主粒子、光晕粒子、碎片粒子
- 颜色源：封面取色、手动主色、手动辅色
- 律动：低频强度、人声强度、节拍跳动
- 镜头：静态、缓慢旋转、节拍推近
- 文字：歌词位置、歌词发光、个人文案

### 3. 安全参数

用户可调参数必须有范围限制：

- 粒子密度
- 扩散范围
- 层次深度
- 音频跟随强度
- 粒子透明度
- 光晕大小
- 镜头晃动
- 动画速度

这些参数继续走现有 `fx` 配置体系，避免第一阶段大改底层架构。

## 不做什么

第一阶段明确不做：

- 不开放任意 JavaScript 执行。
- 不允许用户直接粘贴完整 shader 代码。
- 不一次性重构整个 `public/index.html`。
- 不移除现有 `fx` 和 `mineradio-user-fx-archives-v1` 兼容逻辑。
- 不把上传图片传到外部服务器；图片应保存在本地浏览器存储或本地文件链路里。

## 推荐实现路径

### 第一步：作品化

先把当前存档入口改成“我的作品”，补齐作品名、说明、预览、复制、导入、导出。

成功标准：

- 老用户已有存档还能读取。
- 新作品能保存和恢复。
- 导入无效 JSON 时不会覆盖当前配置。

### 第二步：积木化面板

把 DIY 面板整理成几个清晰分组：

- 作品
- 素材
- 形态
- 粒子
- 律动
- 镜头
- 歌词与文字
- 性能

成功标准：

- 用户能不理解 shader，也能组合出明显不同的效果。
- 不同形态之间切换不会丢失当前作品名和素材。

### 第三步：声明式配方

新增 `creativeRecipe` 字段，用 JSON 描述作品，不执行用户代码。

建议结构：

```json
{
  "type": "mineradio-visual",
  "version": 1,
  "name": "My Visual Work",
  "material": {
    "source": "current-cover"
  },
  "shape": {
    "type": "disc-field"
  },
  "layers": [
    {
      "type": "particles",
      "density": 0.62,
      "colorSource": "cover"
    }
  ],
  "motion": {
    "audioFollow": 0.58,
    "cameraMode": "slow-orbit"
  }
}
```

成功标准：

- 导入前显示预览和来源说明。
- 导入失败有明确提示。
- 超出范围的参数会被拒绝或钳制到安全范围。

## 交互原则

- 普通用户看到的是“积木”和“作品”，不是技术参数堆叠。
- 高级用户可以导入导出 JSON，但仍不能执行代码。
- 上传图片是素材来源，不替代当前专辑封面默认逻辑。
- 所有改动优先兼容现有视觉预设，不把已有好看的效果改坏。

## 验证方式

后续实现时至少验证：

- `git diff --check`
- `node --check server.js`
- 前端内联脚本语法检查
- 浏览器打开播放器主页面
- 浏览器打开 DIY 面板并保存、恢复一个作品
- 导入错误 JSON 时不覆盖现有作品

## 当前结论

已确认采用“模块化 DIY 沙盒”方向。下一步可以进入第一阶段实现：先做“我的作品”和基础积木面板，再逐步接入更自由的粒子形态。
