# 落雪自定义音源兼容层设计

## 目标与定位

为 Mineradio Extended 增加对落雪音乐自定义音源脚本 API 2.0.0 的兼容能力，让用户可以自行导入可信的 `.js` 音源脚本，在官方播放地址因技术原因不可用时尝试解析播放地址。

这项功能是可选的播放地址兼容层，不是新增一个完整音乐平台。它不提供落雪账号、平台登录、搜索、歌单、收藏、评论或会员能力，也不改变 Mineradio 已有平台的内容来源和账号体系。

## 第一阶段范围

- 导入、校验、查看、启用、停用、替换和删除自定义音源脚本。
- 同一时间只允许启用一个音源脚本，避免解析结果和错误来源不明确。
- 兼容落雪自定义音源 API 2.0.0 的 `inited`、`request`、`updateAlert` 事件，以及 `musicUrl` 请求。
- 支持脚本声明和使用 `128k`、`320k`、`flac`、`flac24bit` 音质。
- 将当前播放器歌曲信息转换为落雪脚本所需的最小歌曲元数据。
- 支持以下平台映射：

| Mineradio 平台 | 落雪平台键 | 第一阶段状态 |
| --- | --- | --- |
| 网易云音乐 `netease` | `wy` | 支持 |
| QQ 音乐 `qq` | `tx` | 支持 |
| 酷狗概念版 `kugou` | `kg` | 支持 |
| 酷狗音乐 `kugouMusic` | `kg` | 支持 |
| 汽水音乐 `qishui` | 无标准键 | 不支持 |

- 在播放器中提供独立的“第三方音源”管理弹窗，展示脚本名称、版本、作者、支持平台、音质和当前状态。
- 显示清楚的风险提示：脚本可以将歌曲元数据发送到网络，只应导入来源可信、内容可审查的脚本。

## 非目标

- 不内置、推荐、托管、自动下载或自动更新任何第三方音源脚本。
- 不提供音源市场、脚本分享站或绕过会员限制的说明。
- 不向脚本传递用户 Cookie、登录凭证、Token、账号信息、本地文件路径或播放历史。
- 不让脚本访问 Node.js、Electron 主进程、文件系统、局域网或本机服务。
- 不把第三方音源结果伪装成对应平台的官方播放结果。
- 不在第一阶段实现汽水音乐兼容、歌词、封面或搜索解析。

## 播放顺序与权限边界

播放链路保持“官方接口优先”：

1. 按歌曲所属平台和用户当前登录状态请求官方播放地址。
2. 官方地址可用时直接播放，不调用第三方音源。
3. 用户明确启用了第三方音源，且官方请求属于网络、地址失效、格式不兼容等技术失败时，才允许调用兼容层。
4. 官方接口明确返回 `vip_required`、`paid_required`、需购买、无版权或其他权限限制时，不调用第三方音源。
5. 第三方解析失败后沿用现有可解释的错误提示，不无限重试，也不静默切换平台。

界面和日志必须标明本次播放是否使用了第三方音源，避免用户误认为结果来自官方会员链路。

## 架构

### CustomSourceStore

- 将脚本和元数据保存在 Electron `userData/custom-sources` 目录，不进入 Git、安装包资源或更新补丁。
- 使用稳定 ID 管理脚本文件，禁止脚本名称直接成为文件路径。
- 导入时限制文件类型、文件大小和数量；第一阶段单文件上限为 1 MB。
- 配置仅记录启用状态、脚本元数据和用户设置，不记录歌曲或账号隐私数据。

### CustomSourceRuntime

- 每个已启用脚本在独立、隐藏的 Electron `BrowserWindow` 中运行。
- 使用 `sandbox: true`、`contextIsolation: true`、`nodeIntegration: false`、`nodeIntegrationInWorker: false` 和独立的非持久化 partition。
- 通过受限 preload 只暴露落雪兼容 API；脚本不能直接获得 Electron、Node.js、DOM 中的应用数据或主窗口对象。
- 运行页使用严格 CSP，并在独立 session 中拦截 `fetch`、XHR、WebSocket、图片信标等直接联网方式；脚本只能通过受控的 `lx.request` 访问网络。
- 禁止新窗口、导航、下载、权限请求、媒体采集和生产环境 DevTools。
- 所有 IPC 使用专用通道，并校验发送方 WebContents 和运行时 ID。
- 解析请求设置超时，超时或脚本异常时销毁并按需重建运行时。

### NetworkBroker

- 脚本的 `lx.request` 必须经过主进程网络代理，不允许脚本自行建立任意网络连接。
- 仅允许 `http:` 和 `https:`，默认要求远程公网地址。
- 拒绝回环、私网、链路本地、多播、保留地址、`.local` 域名，以及 `file:`、`data:`、自定义协议。
- 每次请求前解析 DNS 并校验全部返回地址，实际连接固定到已校验的地址，避免校验后再次解析造成 DNS rebinding；每一次重定向都重新校验目标，最多允许 5 次重定向。
- 限制单运行时并发请求数、请求体大小、请求头大小和响应体大小；超限立即终止并返回明确错误。
- 对脚本返回的最终音频 URL 执行相同的协议、DNS 和远程地址校验。
- 日志对 `Authorization`、`Cookie`、Token、Secret、API Key 等字段脱敏，不记录完整查询密钥。

### MusicInfoAdapter

- 只提供解析播放地址所需的歌曲 ID、平台、名称、歌手、专辑、可用音质等最小字段。
- 网易云、QQ 和两个酷狗入口分别映射到 `wy`、`tx`、`kg`。
- 不能可靠映射的平台直接返回“不支持”，不猜测或伪造平台键。

### PlaybackResolver

- 在现有播放流程外增加一个小型解析边界，统一官方播放结果和可选第三方回退。
- 不重写搜索、歌单、收藏、歌词、评论或平台登录模块。
- 解析结果包含来源、音质、URL 和失败原因，供播放层和界面明确展示。
- 第三方功能关闭、脚本不可用或平台不支持时，行为与当前版本保持一致。

## 管理界面

- 在现有顶部工具区增加插头图标按钮，悬停提示“第三方音源”。
- 弹窗沿用当前播放器的深色玻璃风格，不新增独立页面。
- 空状态说明用途、限制和风险，并提供“导入脚本”命令。
- 脚本列表支持启用/停用、替换、删除和查看错误；正在使用的脚本有明确状态标识。
- 导入前再次提示：第三方脚本并非 Mineradio 或原作者提供，脚本可能访问外部服务，用户需自行确认来源。
- 不显示在线推荐、不提供一键下载地址，也不使用暗示破解、解锁会员的文案。

## 错误与降级

- 脚本格式错误、API 版本不兼容、初始化失败、请求超时、网络策略拒绝和返回地址无效分别显示可理解的错误。
- 单次第三方解析失败不得影响官方平台后续播放，也不得使播放器主进程退出。
- 连续失败达到阈值后暂停当前脚本并提示用户检查或停用，避免每首歌重复超时。
- 删除或替换当前启用脚本时先销毁运行时，再更新存储，避免旧脚本继续执行。

## 验收标准

- 可以导入符合 API 2.0.0 的测试脚本，并正确显示脚本元数据、平台和音质。
- 启用、停用、替换和删除脚本后，磁盘状态与运行时状态一致。
- 网易云、QQ、酷狗概念版和酷狗音乐歌曲能转换为正确的落雪平台键；汽水音乐明确显示不支持。
- 官方播放地址成功时不触发第三方解析。
- 技术性失败且用户已启用脚本时，可以调用测试脚本并播放返回的合法远程 URL。
- VIP、付费、购买和版权限制不会触发第三方回退。
- 本机、局域网、链路本地地址和通过重定向或 DNS 解析得到的私网地址均被拒绝。
- 脚本无法读取 Node.js、Electron、用户 Cookie、登录态和本地文件。
- 运行时崩溃、超时或返回异常数据时，播放器仍能继续使用官方链路。
- 自动化测试覆盖存储、协议校验、歌曲映射、网络策略、运行时管理、日志脱敏和播放回退策略。
- 使用 Electron 隐藏窗口完成一次运行时冒烟测试，并在开发版播放器中手动验证管理弹窗和一次官方优先播放。

## 预计改动范围

- `desktop/`：自定义音源存储、沙箱运行时、受限 preload、网络代理和主进程生命周期接入。
- `server.js`：只增加受控的本地解析接口与 resolver 注入，不暴露脚本文件内容。
- `public/index.html`：增加管理弹窗、状态反馈和播放解析边界。
- `test/custom-source/`：使用本地 fixture 脚本覆盖协议、安全和播放策略；不包含真实第三方音源。
- `README.md` 与相关文档：说明功能定位、导入方式、风险、限制和开源致谢。
